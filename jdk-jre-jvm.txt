# Java Architecture – JVM vs JRE vs JDK (Complete Notes)

---

## 1. Core Idea

JDK > JRE > JVM

* JDK = Development Kit
* JRE = Runtime Environment
* JVM = Execution Engine

---

## 2. JVM (Java Virtual Machine)

### Definition

JVM is a virtual machine that executes Java bytecode and converts it into machine-specific instructions.

---

### Responsibilities

* Loads `.class` files (bytecode)
* Verifies bytecode (security)
* Executes code using:

  * Interpreter (line-by-line execution)
  * JIT Compiler (optimizes frequently used code)
* Manages memory:

  * Heap
  * Stack
  * Method Area
* Handles garbage collection

---

### Key Characteristics

* Platform-dependent (different JVM for Windows, Linux, Mac)
* Executes platform-independent bytecode
* Core of Java’s WORA principle

---

### Important Point

JVM **only executes** code — it does not provide Java libraries or full runtime support.

---

### Interview Line

JVM is the runtime engine responsible for executing Java bytecode and converting it into machine-specific instructions.

---

## 3. JRE (Java Runtime Environment)

### Definition

JRE is the environment required to run Java programs. It includes JVM along with standard libraries and runtime support.

---

### Components of JRE

#### 1. JVM

* Execution engine

---

#### 2. Core Java Libraries

These are essential for running any Java program.

Examples:

* java.lang → String, Object, System
* java.util → Collections, List, Map
* java.io → File handling
* java.net → Networking

---

#### 3. Runtime Support Files

* Configuration files
* Security policies
* Charset and timezone data

---

#### 4. Native Libraries

* OS-level integrations (`.dll`, `.so`)
* Used for file system, networking, etc.

---

#### 5. JNI (Java Native Interface)

* Allows Java to interact with native OS code

---

### Purpose

To provide everything needed to **run Java applications**

---

### What JRE does NOT include

* No compiler (`javac`)
* No development/debugging tools

---

### Critical Insight

JVM alone cannot run a program without JRE because:

* It cannot find standard classes (like `System`, `Object`)
* It lacks runtime libraries

---

### Example Error Without JRE

java.lang.NoClassDefFoundError: java/lang/Object

---

### Interview Line

JRE provides the runtime environment required to execute Java programs by including the JVM along with core libraries and supporting files.

---

## 4. JDK (Java Development Kit)

### Definition

JDK is the complete toolkit used to develop, compile, and run Java applications.

---

### Components of JDK

#### 1. JRE

* Includes JVM + libraries

---

#### 2. Development Tools

Important tools:

* javac → Compiler (Java → bytecode)
* java → Runs programs
* jar → Packaging tool
* javadoc → Documentation generator
* jdb → Debugger

---

### Purpose

To provide tools for:

* Writing code
* Compiling code
* Running code

---

### Interview Line

JDK is the complete development kit that includes JRE along with tools like the compiler, debugger, and utilities used to build Java applications.

---

## 5. Relationship Between JVM, JRE, JDK

JDK
└── JRE
└── JVM

---

## 6. Execution Flow (Important)

Step 1: Write code
Hello.java

Step 2: Compile
javac Hello.java → Hello.class (bytecode)

Step 3: Run
java Hello

---

### What happens internally

* JDK → compiles code
* JRE → provides environment
* JVM → executes bytecode

---

## 7. Why JVM Alone is Not Enough

JVM can:

* Execute bytecode

But cannot:

* Provide standard classes (String, System, etc.)
* Handle file I/O, networking, utilities
* Access runtime libraries

---

### Result Without JRE

Program fails with class loading errors

---

## 8. Key Differences

| Feature          | JVM                  | JRE                      | JDK                  |
| ---------------- | -------------------- | ------------------------ | -------------------- |
| Full Form        | Java Virtual Machine | Java Runtime Environment | Java Development Kit |
| Purpose          | Execute bytecode     | Run Java apps            | Develop Java apps    |
| Contains         | Execution engine     | JVM + libraries          | JRE + tools          |
| Compiler present | No                   | No                       | Yes                  |
| Used by          | Runtime              | End users                | Developers           |

---

## 9. Simple Analogy

* JVM = Engine
* JRE = Engine + fuel + wiring
* JDK = Full factory (build + run)

---

## 10. WORA (Write Once Run Anywhere)

Java ensures WORA by:

* Compiling code into platform-independent bytecode
* Running bytecode using JVM on any OS

Each OS has its own JVM implementation, so the same `.class` file runs everywhere.

---

## 11. Final Interview Answer

JVM is the core execution engine that runs Java bytecode. JRE provides the runtime environment by including the JVM along with standard libraries and support files required to run Java programs. JDK is the complete development kit that includes JRE and additional tools like the compiler and debugger used to build Java applications.

---

## 12. Ultra Short Memory Trick

JVM = runs code
JRE = runs + libraries
JDK = develop + run
